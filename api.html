Read

GET/data

The board document: { projects: [ { id, name, color, board: { todo, inprogress, done } } ], view }. Cards are { id, title, desc, notes }. Live stickies only, ordered by position. Same envelope in every mode below; unknown project id returns an empty projects array with 200.

Query paramEffect
?titles=1Cards shrink to { id, title } (~8KB vs ~48KB full)
?project=<id>Only that project's board
?list=1Projects as { id, name, color }, no boards (~0.7KB); wins over other params
?archived=1The archived slice instead of the live one

project, titles, and archived compose: /data?project=kanban-001&archived=1&titles=1.

GET/audit

The audit trail, newest first: [ { id, ts, action, stickyId, old, new } ]. old/new are the full before/after snapshots each op recorded; ts is UTC. Same contract as the MCP audit-query tool.

Query paramEffect
?sticky=<id>Only rows for that sticky — "what happened to kb-t23"
?limit=<n>Max rows (default 20)

Write

POST/op

One operation per request, JSON body, UTF-8. Each op runs as a single transaction and (except view-set) writes an audit row with full old/new snapshots. Success: 200 {"ok":true}. Invalid or failed op: 400 with a plain-text reason.

kindBody
card-add{ kind, projectId, col, card: { id, title, desc, notes } }
card-edit{ kind, cardId, title, desc, notes } — merges: omitted field = unchanged, '' clears
card-move{ kind, cardId, toProjectId, toCol, toIndex }
card-delete{ kind, cardId }
card-archive{ kind, cardId } — card leaves the live board; re-archiving an archived card fails
project-add{ kind, project: { id, name, color } }
project-edit{ kind, projectId, name, color } — merges like card-edit
project-delete{ kind, projectId } — deletes its stickies too; audit row keeps the full snapshot
project-import{ kind, projects: [ { id, name, color, board } ] } — skips existing project ids
view-set{ kind, view } — no audit row

col is one of todo / inprogress / done. Callers supply card/project ids; the readable convention is <prefix>-t## (see kb-t21).

Static

GET/<file>

Serves files from the workshop root (path-traversal guarded, Cache-Control: no-store). / serves the board. Anything else: 404. Non-GET/POST methods: 405.

Examples

# PowerShell - cheap orientation read
Invoke-RestMethod 'http://localhost:8123/data?titles=1'

# PowerShell - what happened to a sticky
Invoke-RestMethod 'http://localhost:8123/audit?sticky=kb-t23'

# PowerShell - add a sticky
$op = @{ kind='card-add'; projectId='kanban-001'; col='todo';
         card=@{ id='kb-t99'; title='Example'; desc=''; notes='' } } | ConvertTo-Json
Invoke-RestMethod http://localhost:8123/op -Method Post -ContentType 'application/json' -Body $op
// JavaScript - what the board itself does
const doc = await (await fetch('/data')).json();
await fetch('/op', { method: 'POST', headers: { 'Content-Type': 'application/json' },
                     body: JSON.stringify({ kind: 'view-set', view: 'all' }) });

Not HTTP

Server down? Dot-source stickies-db.ps1 (pwsh) and call the same layer directly: Open-StickiesDb, Invoke-StickiesOp, Export-StickiesJson (same filters as switches: -ProjectId -TitlesOnly -ProjectList -Archived), Export-StickiesAuditJson (-StickyId -Limit).
